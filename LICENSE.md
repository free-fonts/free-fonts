## 🚨 Troubleshooting & Common Issues

### Fonts not appearing in apps after install
- Restart the application (e.g., Photoshop, Word) — some apps cache font lists
- Run `fc-cache -fv` on Linux (if using WSL) or restart your Mac/PC
- Check that the script’s `AutoInstall` is actually enabled and the download completed without errors

### Script GUI doesn’t launch on Mac
- Ensure you right‑clicked → **Open** the first time to bypass Gatekeeper
- If it still fails, run `xattr -cr /Applications/FontScript.app` in Terminal and try again
- Make sure your macOS version is supported (macOS 11+)

### Download speed is very slow
- Reduce `MaxConcurrentDL` to 2–3 if your connection is unstable
- Switch from Wi‑Fi to a wired connection for large syncs
- The catalog metadata (initial sync) can be large; let it run overnight if needed

### License filter shows commercial‑safe fonts that are actually personal use only
- Font metadata sometimes contains errors. Use the “Report a Font” button inside the script to flag it
- Double‑check the font’s homepage or README before using in a client project

### Auto‑install fails on Windows with “permission denied”
- Run the script loader as **Administrator** (right‑click → Run as Administrator)
- Ensure your download path isn’t inside a protected folder (avoid `C:\Windows\Fonts` directly; the script copies to the proper location automatically)
